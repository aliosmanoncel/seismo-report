ABSTRACT EQ-LiveTürkiye is an open-source seismic monitoring platform combining a real-time earthquake feed with a scientifically rigorous Gutenberg-Richter b-value hazard layer for Turkey. The platform integrates a 126-year composite catalog assembled from ISC (1900–1997) and EMSC (1998–2026) data using period-specific completeness thresholds (staircase method). Magnitude homogenization follows Scordilis (2006) empirical relations (ML/MD/mb → Mw). The b-value grid was computed using Aki (1965) maximum likelihood estimation with Utsu (1966) binomial correction, following the Fixed Area Method of Öncel & Wyss (2000): R=100 km, 10-km sliding step, N≥50. Turkey-wide b=1.031 is consistent with published literature. The platform is publicly accessible via GitHub Pages and built entirely on open-source tools.

🌐 1. Platform Overview — What EQ-LiveTürkiye Shows
Purpose

EQ-LiveTürkiye is a browser-based seismic monitoring and hazard visualization platform for Turkey. It combines two complementary data streams: (1) a real-time EMSC earthquake feed that updates automatically with recent events, and (2) a pre-computed Gutenberg-Richter b-value grid derived from a 126-year composite catalog. Both layers are overlaid on an interactive Leaflet.js map, enabling simultaneous assessment of ongoing seismicity and long-term seismic hazard patterns.

Step 1 — FDSN Data Retrieval

Both ISC and EMSC provide FDSN-compliant web service APIs (fdsnws/event/1/query). The Python script fetch_combined_catalog.py queries each service in 5–10 year chunks to avoid API result limits (ISC: 20 000 events/query; EMSC: 20 000 events/query). The geographic bounding box is set to encompass Turkey and its near-field: 33–45°N, 23–48°E.

⚙️ Retrieval parameters

  • Format: text (pipe-delimited, lightweight)
  • Ordering: time-asc per chunk
  • Rate limiting: 1.5 s sleep between requests
  • Retry: 2× delay on network error
  • Chunk size: 10 yr (ISC) / 5 yr (EMSC)
Step 2 — Staircase Completeness Merge

Seismic network density has evolved over time; a single magnitude threshold applied across the full 126-year record would either exclude large numbers of smaller events or include incomplete earlier periods. The staircase approach addresses this by applying the minimum completeness magnitude (Mc) appropriate for each time period, then taking the union of all sets.

Step 3 — Duplicate Removal

Events present in both ISC and EMSC catalogs (especially near the 1998 overlap boundary) are identified by temporal and spatial proximity. Duplicates within 60 seconds and 50 km of each other are merged, retaining the higher-Mw estimate. This removes 396 duplicate pairs, yielding 83 918 unique events.

⚖️ 3. Magnitude Homogenization — Scordilis (2006)
Why Homogenization Is Necessary

The composite ISC+EMSC catalog contains events reported in at least eight different magnitude scales (MD, ML, mb, Ms, Mw and variants). Mixing scales introduces systematic bias: a body-wave magnitude mb=5.0 is not equivalent to a moment magnitude Mw=5.0. For probabilistic seismic hazard analysis (PSHA) all magnitudes must be expressed on a single, physically meaningful scale — moment magnitude Mw.

Scordilis (2006) Conversion Relations

Scordilis (2006) derived empirical global relations for converting the most common magnitude scales to Mw using a global dataset of ~20 000 earthquakes with known moment tensor solutions. These relations are applied in order of reliability.

mb → Mw (body-wave magnitude)

Mw = 0.85·mb + 1.03   (3.5 ≤ mb ≤ 6.2)

Valid for teleseismic body-wave magnitudes from ISC global station network.

ML / MD → Mw (local / coda magnitude)

Mw = 0.0376·M² + 0.646·M − 0.269   (1.0 ≤ M ≤ 6.5)

Quadratic relation accounts for the non-linear saturation of local magnitude at higher values. Applied to ML, MD, md, and mL.

Ms → Mw (surface-wave magnitude)

Mw = 0.646·Ms + 2.079   (3.0 ≤ Ms ≤ 6.1)
Mw = 0.994·Ms + 0.115   (Ms > 6.1)

Two-segment relation reflects saturation of Ms for large earthquakes.

Critical Note: Scordilis Bias in b-value Computation

⚠️ Known Issue: ML=3.0 → Mw≈2.0

  • Scordilis ML→Mw: for ML=3.0 → Mw = 0.0376×9 + 0.646×3 − 0.269 = 2.007
  • Filtering by converted Mw≥3.0 drops ~66 000 EMSC events (MD/ML ≈ 3.0–4.5)
  • This inflates mean(Mw) from ~2.5 to ~4.2, producing artificially low b-values (~0.28)
  • Fix: b-value grid computed using original reported mag, not converted Mw
  • Completeness filtering also uses original mag ≥ Mc (not converted Mw)
The converted Mw field is retained in eq_historical.json for other uses (PSHA source models, moment rate calculations). Only the b-value computation bypasses it to avoid the conversion-induced bias.
📊 4. Gutenberg-Richter b-value — Aki (1965) + Öncel & Wyss (2000)
Gutenberg-Richter Frequency-Magnitude Relation
log₁₀(N) = a − b·M

The b-value slope describes the relative proportion of large to small earthquakes. A b≈1 (global average) means each unit increase in magnitude corresponds to a 10-fold decrease in frequency. Lateral variations in b reflect differences in stress state, material heterogeneity, and temperature.

Aki (1965) Maximum Likelihood Estimation
b = log₁₀(e) / (M̄ − Mc,eff)

The MLE is statistically superior to least-squares regression because it makes no assumption about the distribution of the dependent variable and produces unbiased estimates. Here M̄ is the sample mean magnitude above Mc.

Utsu (1966) Binomial Correction
Mc,eff = Mc − ΔM = 3.0 − 0.05 = 2.95

Earthquake catalogs typically report magnitudes rounded to the nearest 0.1 unit. Utsu (1966) showed that this discretization introduces a systematic underestimate of b; the correction is ΔM = bin-width/2 = 0.05, yielding an effective completeness threshold of Mc,eff = 2.95.

Fixed Area Method — Öncel & Wyss (2000)
Reference: Öncel, A. O. & Wyss, M. (2000). The major asperities of the 1999 Mw 7.4 İzmit earthquake defined by the microseismicity of the two decades before it. Geophysical Journal International, 143(2), 501–506.

Rather than a moving window with variable area, the Fixed Area Method samples events within a constant-radius circle centered on each grid node. This ensures the spatial extent of the sample remains consistent across nodes, eliminating area-induced bias in b-value estimation. Öncel & Wyss (2000) applied R=20 km and 5-km step for the İzmit rupture zone; we scale to R=100 km and 10-km step for the Turkey-wide analysis.

Color Normalization

A fixed RdYlGn color scale is applied: b=0.5 → red (high stress), b=1.0 → yellow (normal), b=1.5 → green (low stress / heterogeneous). The normalized weight w = 1 − (b − 0.5) / 1.0 is clipped to [0, 1]. This fixed scale ensures spatial comparability across zoom levels.

Zoom-Adaptive Display

⚙️ Zoom-level sampling strategy

  • Zoom 6–8: every 3rd point (~0.27° ≈ 30 km) — regional view
  • Zoom ≥ 8: all points (0.09° ≈ 10 km) — local detail

Regional Interpretation

🔴 Low b-value zones (b < 0.8) — Elevated stress

Eastern Anatolian Fault Zone (EAFZ), particularly the Kahramanmaraş triple junction area. Low b-values here are consistent with high tectonic loading rates on the left-lateral EAFZ. The February 2023 Mw7.8 and Mw7.7 earthquake sequence epicentral region shows characteristic pre-seismic low-b signature in historical data.

🟢 High b-value zones (b > 1.3) — Heterogeneous / geothermal

Western Anatolia extensional province (Gediz, Büyük Menderes grabens), Central Anatolian volcanic arc, and parts of the Black Sea coastal zone. High b-values in these areas reflect crustal heterogeneity, elevated pore-fluid pressures, or thermally weakened fault zones.

✅ Validation

  • Turkey-wide b = 1.031 (N=76 514) — consistent with published values (Kalafat et al. 2007: b≈1.0–1.1)
  • b range 0.460–2.249, mean 1.167 — physically plausible for tectonically diverse region
  • 2 804 unmapped nodes (N<50) concentrated in offshore Black Sea and low-seismicity Central Anatolia
🔶 6. PSHA Integration — GEM OpenQuake Engine
From b-value Grid to Seismic Source Model

Probabilistic Seismic Hazard Analysis (PSHA) quantifies the probability of exceeding a given ground-motion intensity measure (e.g., PGA, Sa) at a site within a specified time window. The GEM Foundation's OpenQuake Engine is the leading open-source PSHA platform, used in the Global Earthquake Model and national hazard studies worldwide.

In an OpenQuake source model, each area source or point source requires a Gutenberg-Richter MFD (Magnitude Frequency Distribution) defined by an a-value and b-value. The b-value grid produced here provides spatially resolved b-values that can be directly incorporated into OpenQuake source_model.xml files:

OpenQuake source_model.xml — b-value entry example:
<areaSource id="TR_01" name="Marmara Region">
  <truncGutenbergRichterMFD aValue="5.2" bValue="0.95"
    minMag="5.0" maxMag="7.5"/>
</areaSource>

The bValue field is populated from the spatial average of b-value grid cells overlapping each source zone.

⚠️ 7. Limitations and Future Work

⚠️ Current Limitations

  • No declustering: Aftershock sequences (e.g., 2023 Kahramanmaraş swarm: ~10 000 events) inflate b-values in affected regions. Gardner-Knopoff (1974) or Reasenberg (1985) declustering not yet applied.
  • Fixed Mc = 3.0: Local Mc may vary across Turkey. Wiemer & Wyss (2000) MAXC method could provide spatially variable Mc.
  • EMSC only for b-grid: The pre-1998 ISC record is not used in b-value computation due to heterogeneous completeness. Extending to 1965 (Mc=4.5) could improve coverage in low-seismicity areas.
  • No uncertainty bounds: b-value confidence intervals (Shi & Bolt 1982) are computed but not yet displayed on the map.
  • Static grid: The b-value grid is pre-computed; the platform does not yet update it automatically when new EMSC data arrives.

✅ Planned Improvements

  • Gardner-Knopoff declustering for mainshock-only catalog
  • MAXC spatially variable Mc mapping
  • Shi & Bolt (1982) uncertainty bounds on b-value display
  • Automated catalog update via GitHub Actions (monthly refresh)
  • OpenQuake source model XML export script
